Add opt-in recorded-time history capture (F1a). Enable with `createStore(graph, backend, { history: true })` and every mutation of an existing node or edge writes a complete pre-image to history side-tables (`typegraph_node_history` / `typegraph_edge_history`) in the same transaction — `create` writes none. Read per-entity history with `store.nodes.<Kind>.history(id)` / `store.edges.<kind>.history(id)`: typed versions, newest-first, each carrying `{ image, recordedFrom, recordedTo, op, schemaVersion, txId, meta }` where `op` is `update | delete | restore | hardDelete`. Trim retention with `store.history.prune({ before })`, and stamp audit metadata via `store.transaction(fn, { meta })`. Unlike soft delete (which hides rows from all history), `hardDelete` preserves prior history and captures the final image. Off by default with zero overhead — compiled mutation SQL is byte-identical when history is disabled. `backend.capabilities.history` reports `"atomic"` (a history row commits iff its mutation commits — every transactional backend, and Postgres even over neon-http via a single data-modifying CTE) or `"best-effort"` (non-transactional SQLite / Cloudflare D1: mutation-first, capture-second, so a crash loses history but never fabricates a transition). This is versioned state queried as history, not an event log: no offsets, streams, or replay API.
