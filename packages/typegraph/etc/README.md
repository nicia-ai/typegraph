# Public API report

The `*.api.md` files are checked-in API contracts generated by API Extractor.
The runner derives every public entrypoint from the package export map, so a
new subpath cannot silently miss the API gate. Update them intentionally after
reviewing a public API change:

```bash
pnpm --filter @nicia-ai/typegraph api-report:update
```

CI runs `test:api-report` against the declaration build and fails when the
generated report differs from the checked-in contract.

The runner retains forgotten-export definitions because they are required to
describe structural public contracts such as `Store`. It removes API
Extractor's trailing diagnostic appendix, whose source locations contain
content-hashed tsup chunk names and would otherwise turn internal-only edits
into false API changes. The declarations and their inline API diagnostics stay
fully checked.

The existing entrypoints predate this gate and intentionally reference
structural types that are not independently importable. Their exact
`ae-forgotten-export` symbol sets are fingerprinted in `scripts/api-report.ts`:
adding, removing, or renaming one fails verification even if the warning count
does not change. New entrypoints default to zero forgotten exports; `/backend`
establishes that clean authoring contract. Missing release tags and unresolved
links remain suppressed, while every other API Extractor diagnostic and the
full declaration report stay checked.
