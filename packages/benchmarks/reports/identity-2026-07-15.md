Date: 2026-07-15

- Apple M4 Pro, 48 GiB RAM

## Identity-disabled comparison

Values are median milliseconds per operation. Ranges are the medians of three
independent runs where the established lane was repeated.

| Shape | Pre-feature | Final | Assessment |
| --- | ---: | ---: | --- |
| Store construction | 0.001904 | 0.002073 | +0.000169 ms; p95 0.002937 ms |
| Property-only update | 0.0392-0.0412 | 0.039257 | Within baseline range; no identity SQL or lock |
| Single node create | 0.032-0.035 | 0.034-0.035 | Within baseline range |
| Transactional node create | 0.026 | 0.026-0.028 | Within run noise |
| Bulk node create | 0.015-0.016 | 0.016 | Within baseline range |
| Interchange import | 0.010-0.011 | 0.010-0.011 | Unchanged |
| Cached ordinary query | 0.028-0.029 | 0.029-0.033 | Overlapping run distributions |
| Prepared ordinary query | 0.025-0.026 | 0.026 | Within baseline range |

Identity-disabled graphs take no identity lock, folding probe, closure write, or
query expansion. The implementation also avoids async identity-hook yields,
tombstone probes, and write-session lookup when the corresponding capabilities
are disabled. Store construction remains about two microseconds; its median
shift is 0.169 microseconds. No disabled write, import,
or ordinary-query lane shows a practically meaningful regression.

## Folding probe and index decision

| Shape | SQLite median / p95 | PostgreSQL median / p95 |
| --- | ---: | ---: |
| Existing per-kind primary-key probes | 0.006308 / 0.006620 | 0.584718 / 0.658721 |
| Global probe with temporary `(graph_id, id)` index | 0.003067 / 0.004077 | 0.185438 / 0.209788 |
| Disabled create before temporary index | 0.028842 / 0.031525 | 0.913971 / 0.963023 |
| Disabled create after temporary index | 0.031554 / 0.041529 | 0.924685 / 1.027248 |

Decision: do not add a permanent global nodes `(graph_id, id)` index. It saves
about 0.0032 ms per SQLite fold probe and 0.40 ms per PostgreSQL probe, but the
temporary-index comparison increased disabled-create median and p95 on both
backends. Runtime folding now uses the measured per-kind primary-key probes and
entity-scoped closure maintenance; the final folded-create lane improved to
0.153 ms on SQLite and 4.279 ms on PostgreSQL without taxing identity-disabled
graphs with another global index.
